# b2p v2 — Phase P1e Implementation Plan (CLI wiring — completes P1)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire P1a–P1d into the real CLI so `b2p receive` / `b2p send` default to the new PAKE + ntfy + WebRTC stack (design §6, §8), with `--tunnel` restoring the P0 path and `--rendezvous` overriding the signaling host. This is the last P1 phase — after it, the headline v2 flow works end to end.

**Architecture:** A testable orchestration layer `src/session.rs` composes the pieces — `receive_p1` runs `handshake(Receiver) → connect(Receiver) → stream::recv_into`; `send_p1` runs `handshake(Sender) → connect(Sender) → stream::send_source` — parameterized on a `Rendezvous` and STUN list so it's tested offline (MemRendezvous + loopback + a real transfer, running the *actual handshake* this time). `stream::recv_into` gains an accept **callback** (replacing `auto_accept`/`overwrite`) so the CLI can prompt interactively and close the P1c folder-clobber gap. `main.rs` stays thin: parse args, dispatch by code form (`parse_any`), build the ntfy rendezvous, call `session::*` with real STUN, render progress + the interactive prompt + a `via WebRTC (STUN)` transparency line, and run `b2p doctor` on connect failure.

**Tech Stack:** Rust 2021, existing `rvcode`/`pake`/`rendezvous`/`handshake`/`transport`/`stream`/`doctor` modules, clap. No new external deps.

## Global Constraints

- `cargo test` stays fully **offline**: `session.rs` is tested with MemRendezvous + loopback (no STUN, no ntfy); `main.rs` gets unit tests for `parse_any` and the accept-decision helper only. The real ntfy/STUN path is a manual smoke, not a `cargo test`.
- CI gate: `cargo fmt --check`, **`cargo clippy --all-targets -- -D warnings`** (note `--all-targets` — CI lints `#[cfg(test)]` code; use this exact command in every gate step), `cargo test` — all green.
- cargo is not on PATH: prefix every cargo command with `export PATH="$HOME/.cargo/bin:$PATH"`.
- **One transfer per SessionKey** (P1c invariant): `receive_p1`/`send_p1` run exactly one handshake and one transfer; no retry-on-cached-key. A retry means a fresh `handshake`.
- Default behavior flips: without `--tunnel`, `receive`/`send` use the P1 stack. `--tunnel` (and v1 `https://…` codes on `send`) use the unchanged P0 path.
- `receive` prints the **human code only** (e.g. `7-otter-zebra`). NOTE: the human code cannot also be rendered as a `b2p://` URL — its topic is channel-derived and its secret is 3 bytes, while the URL form requires an independent 16-byte topic+secret (P1a security model). The `b2p://` power-user form is out of scope for P1e.
- STUN servers (design §4.2), scheme-prefixed for `RTCIceServer`: `["stun:stun.l.google.com:19302", "stun:stun.cloudflare.com:3478"]`.
- Consumed interfaces (exact, on main): `rvcode::{parse, is_rendezvous_code, RendezvousCode, PakeSecret}`; `pake::Role`; `rendezvous::{Rendezvous, ntfy::NtfyRendezvous}`; `handshake::handshake(rv: &dyn Rendezvous, topic, password: &[u8], role) -> Result<SessionKey>`; `transport::webrtc::connect(rv: Arc<dyn Rendezvous>, topic, key, role, stun: &[String], timeout) -> Result<WebRtcChannel>`; `stream::{send_source, recv_into, MsgChannel}`; `archive::Source`; `code::Code` + `send::send` (P0 tunnel path); `doctor`.

    use crate::archive;
    use crate::rendezvous::Rendezvous;
    use async_trait::async_trait;
    use futures::stream::{BoxStream, StreamExt};
    use std::sync::{Arc, Mutex};
    use std::time::Duration;

    // Same in-memory rendezvous as handshake.rs / transport tests (history replay).
    struct MemRendezvous {
        log: Mutex<Vec<(String, Vec<u8>)>>,
        tx: tokio::sync::broadcast::Sender<(String, Vec<u8>)>,
    }
    impl MemRendezvous {
        fn new() -> Arc<Self> {
            Arc::new(MemRendezvous { log: Mutex::new(vec![]), tx: tokio::sync::broadcast::channel(1024).0 })
        }
    }
    #[async_trait]
    impl Rendezvous for MemRendezvous {
        async fn publish(&self, topic: &str, frame: &[u8]) -> anyhow::Result<()> {
            self.log.lock().unwrap().push((topic.to_string(), frame.to_vec()));
            let _ = self.tx.send((topic.to_string(), frame.to_vec()));
            Ok(())
        }
        async fn subscribe(&self, topic: &str) -> anyhow::Result<BoxStream<'static, Vec<u8>>> {
            let topic = topic.to_string();
            let rx = self.tx.subscribe();
            let history: Vec<Vec<u8>> = self.log.lock().unwrap().iter().filter(|(t, _)| *t == topic).map(|(_, f)| f.clone()).collect();
            let live = futures::stream::unfold((rx, topic), move |(mut rx, topic)| async move {
                loop {
                    match rx.recv().await {
                        Ok((t, f)) if t == topic => return Some((f, (rx, topic))),
                        Ok(_) => continue,
                        Err(tokio::sync::broadcast::error::RecvError::Lagged(_)) => continue,
                        Err(_) => return None,
                    }
                }
            });
            Ok(futures::stream::iter(history).chain(live).boxed())
        }
        async fn close(&self) -> anyhow::Result<()> { Ok(()) }
    }

    #[tokio::test(flavor = "multi_thread", worker_threads = 2)]
    async fn full_p1_stack_file_transfer() {

```rust
//! P1 session orchestration: run the PAKE handshake, form the WebRTC
//! transport, and move the payload — the composition `b2p receive`/`send`
//! use by default (design §5). Parameterized on a `Rendezvous` + STUN list
//! so it runs offline in tests (MemRendezvous + loopback). One handshake +
//! one transfer per call — a SessionKey is single-use (see stream::stream_key).

use crate::archive::Source;
use crate::handshake::handshake;
use crate::pake::Role;
use crate::protocol::Manifest;
use crate::rendezvous::Rendezvous;
use crate::stream::{recv_into, send_source, MsgChannel};
use crate::transport::webrtc::connect;
use std::path::Path;
use std::sync::Arc;
use std::time::Duration;

pub async fn receive_p1(
    rv: Arc<dyn Rendezvous>,
    topic: &str,
    secret: &[u8],
    out_dir: &Path,
    accept: impl FnOnce(&Manifest) -> bool,
    stun: &[String],
    timeout: Duration,
    progress: Option<indicatif::ProgressBar>,
) -> anyhow::Result<String> {
    let key = handshake(rv.as_ref(), topic, secret, Role::Receiver).await?;
    let mut ch = connect(rv.clone(), topic, &key, Role::Receiver, stun, timeout).await?;
    recv_into(&mut ch, &key, out_dir, accept, progress).await
}

pub async fn send_p1(
    rv: Arc<dyn Rendezvous>,
    topic: &str,
    secret: &[u8],
    source: &Source,
    stun: &[String],
    timeout: Duration,
    progress: Option<indicatif::ProgressBar>,
) -> anyhow::Result<String> {
    let key = handshake(rv.as_ref(), topic, secret, Role::Sender).await?;
    let mut ch = connect(rv.clone(), topic, &key, Role::Sender, stun, timeout).await?;
    send_source(&mut ch, &key, source, progress).await
}
```

Note: `MsgChannel` is imported so `&mut ch` coerces to `&mut dyn MsgChannel` at the `recv_into`/`send_source` call (they take `&mut dyn MsgChannel`; `ch` is a concrete `WebRtcChannel`, so pass `&mut ch`).

use std::sync::Arc;
use std::time::Duration;

const STUN_SERVERS: [&str; 2] = ["stun:stun.l.google.com:19302", "stun:stun.cloudflare.com:3478"];
const DEFAULT_RENDEZVOUS: &str = "https://ntfy.sh";
const CONNECT_TIMEOUT: Duration = Duration::from_secs(20);

/// Decide whether to accept an incoming transfer. `--yes` skips the prompt
/// (still refusing a Kind::File clobber unless `--overwrite`); otherwise it
/// prints a summary and reads y/N from stdin.
fn accept_decision(m: &b2p::protocol::Manifest, out_dir: &std::path::Path, yes: bool, overwrite: bool) -> bool {
    use b2p::protocol::Kind;
    let clobber = m.kind == Kind::File
        && out_dir.join(std::path::Path::new(&m.name).file_name().map(|n| n.to_string_lossy().to_string()).unwrap_or_default()).exists();
    if yes {
        return !(clobber && !overwrite);
    }
    let summary = match m.kind {
        Kind::Text => "Incoming text snippet".to_string(),
        _ => format!("Incoming: {} file(s), {} bytes total", m.entries.len(), m.total_size),
    };
    eprintln!("\n{summary}");
    if clobber {
        eprintln!("  WARNING: destination file exists and will be overwritten");
    }
    eprint!("Accept? [y/N] ");
    use std::io::Write;
    let _ = std::io::stderr().flush();
    let mut line = String::new();
    if std::io::stdin().read_line(&mut line).is_err() {
        return false;
    }
    line.trim().eq_ignore_ascii_case("y")
}

    out: PathBuf,
    yes: bool,
    overwrite: bool,
    rendezvous: Option<String>,
    tls: &TlsOpts,

    let rv: Arc<dyn b2p::rendezvous::Rendezvous> =

async fn do_send(
    code: String,
    paths: Vec<PathBuf>,
    text: Option<String>,
    rendezvous: Option<String>,
    tls: &TlsOpts,
) -> anyhow::Result<()> {
    let source = match &text {
        Some(t) => archive::prepare_text(t),
        None => { eprintln!("Preparing..."); archive::prepare(&paths)? }
    };
    if b2p::rvcode::is_rendezvous_code(&code) {
        let rc = b2p::rvcode::parse(&code).context("invalid code")?;
        let base = rendezvous.as_deref().unwrap_or(DEFAULT_RENDEZVOUS);
        let rv: Arc<dyn b2p::rendezvous::Rendezvous> = Arc::new(b2p::rendezvous::ntfy::NtfyRendezvous::new(base, tls)?);
        let stun: Vec<String> = STUN_SERVERS.iter().map(|s| s.to_string()).collect();

        let desc = match b2p::session::send_p1(rv, &rc.topic, &rc.secret.0, &source, &stun, CONNECT_TIMEOUT, None).await {
            Ok(d) => d,
            Err(e) => return connect_failed(e, &rc.topic, tls).await,
        };
        eprintln!("via WebRTC (STUN)");
        eprintln!("Done: {desc}");
        Ok(())
    } else {

        if let Some(b) = bar { b.finish(); }
        eprintln!("Done: {desc}");
        Ok(())
    }
}
